# 아이디에이션 후보 3개 비교표 (Scorecard)

## 1) 목적
- 겹쳐진 프로젝트 아이디어를 같은 기준으로 비교해, "지금 당장 어떤 시나리오를 우선 발전시킬지" 결정한다.
- 기준은 `IDEATION_ONEPAGER.md`의 Go/No-Go 프레임을 사용한다.

## 2) 평가 기준과 가중치
| 기준 | 설명 | 가중치 |
|---|---|---|
| 업무가치 | 반복업무/장애대응 시간을 실제로 줄이는 정도 | 20 |
| 실행가능성 | 문서가 아닌 실제 액션(티켓/작업 반영)까지 연결 가능성 | 20 |
| 보안통제성 | 최소권한/승인게이트/차단정책 적용 용이성 | 20 |
| 감사가능성 | actor/task/evidence 로그로 사후 추적 가능한 정도 | 15 |
| 조직적합성 | 기존 팀 프로세스(Redmine, 운영체계)와의 접점 | 15 |
| 운영안정성 | 실패복구/승인대기/롤백 경로 설계 용이성 | 10 |

점수 규칙:
- 항목별 1~5점 (5점 최고)
- 총점 = 항목점수 x 가중치의 합

## 3) 후보 시나리오 정의
| 후보 ID | 아이디어명 | 요약 |
|---|---|---|
| S1 | 운영장애 대응 오케스트레이션 | 장애 신호 + RAG 근거 + 승인 + Redmine MCP 실행 |
| S2 | 배포/변경 위험 사전심사 에이전트 | 릴리즈 전 로그/변경이력/RAG를 기반으로 위험점수와 승인권고 생성 |
| S3 | 온콜 핸드오버 자동화 에이전트 | 교대 시점 이슈/로그/진행상황을 요약하고 인수인계 태스크 생성 |

## 5) 우선순위 결론
1. S1 운영장애 대응 오케스트레이션 (470점)
2. S2 배포/변경 위험 사전심사 에이전트 (400점)
3. S3 온콜 핸드오버 자동화 에이전트 (400점)

결론:
- 1순위는 S1이다.
- S2/S3는 S1에서 만든 공통 자산(승인/정책/감사/RAG 연결)을 재사용해 후속 확장한다.

## 6) 시나리오별 리스크 한 줄 요약

|---|---|---|
| S1 | 고위험 액션의 과자동화 | 승인 필수 액션 분리 + Dry-run 기본 |

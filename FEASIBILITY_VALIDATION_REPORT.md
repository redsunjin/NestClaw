# 실행가능성 검증보고서 (비IT 업무자/기업 적용)

## 1) 검증 질문
- 현재 접근(로컬 업무 위임 오케스트레이션)이
  - 비IT 업무자에게 실제로 사용 가능한가?
  - 회사 단위 에이전트 운영 모델로 확장 가능한가?

검증 기준은 `AGENT_EXPERT_GROUP.md`의 전문가 그룹 역할과 `REVIEW_REPORT.md`의 가드레일을 따른다.

## 2) 검증 프레임
### 2.1 평가 축
- 사용 난이도: 비IT 사용자가 프롬프트 엔지니어링 없이 사용할 수 있는가
- 운영 안정성: 실패/차단/승인 흐름이 예측 가능한가
- 보안/준수성: 회사 적용 시 권한/로그/정책이 감사 가능한가
- 가치 실현성: 반복 업무 자동화 효과가 명확한가

### 2.2 판정 스케일
- High: 즉시 적용 가능
- Medium: 조건 충족 시 적용 가능
- Low: 구조 변경 없이는 적용 어려움

## 3) 전문가 그룹 합동 판정
| Agent | 비IT 업무자 적용성 | 회사 적용성 | 핵심 판단 근거 |
|---|---|---|---|
| A01 Product Owner | Medium | High | 업무 범위 고정 시 가치 전달이 명확함 |
| A02 Workflow Engineer | Medium | High | 상태머신/재시도 설계 시 운영 가능 |
| A03 LLM Orchestrator | Medium | Medium | 체인 단순화 시 품질 확보 가능 |
| A04 Security Privacy | Medium | High | RBAC/화이트리스트/차단 룰로 통제 가능 |
| A05 UX Operations | High | Medium | 템플릿+버튼형 UX면 비IT 진입 장벽 낮음 |
| A06 QA Reliability | Medium | High | 회귀 시나리오 고정 시 회사 운영 적합 |
| A07 Compliance Advisor | Medium | Medium | 로그 보존 정책 문서화가 전제 |
| A08 Domain SME | High | High | 도메인 검수 기준 있으면 품질 수용 가능 |

## 4) 종합 결론
- 비IT 업무자 대상: **조건부 가능 (Medium -> High 전환 가능)**
- 회사 대상: **실행 가능 (High, 가드레일 전제)**

핵심은 기술 난이도가 아니라 운영 설계다.  
즉, "에이전트 성능"보다 "승인/권한/로그/템플릿 UX" 완성도가 채택을 좌우한다.

## 5) 비IT 업무자 적용 조건 (필수)
1. 프롬프트 입력 최소화
- 자유 입력보다 업무 템플릿 기반(예: 회의요약, 보고서 초안, 일정정리)

2. 버튼 중심 실행
- `생성 -> 실행 -> 상태 -> 결과 확인`만 노출

3. 실패 시 자동 안내
- 실패 원인, 다음 행동, 승인 필요 여부를 자연어로 안내

4. 안전 기본값
- 개인 데이터 경로 기본 차단
- 위험 액션은 항상 승인 필요

## 6) 회사 적용 조건 (필수)
1. 역할 기반 접근(RBAC)
- 요청자/검토자/승인자/관리자 권한 분리

2. 감사 로그 체계
- 누가, 언제, 어떤 입력으로, 어떤 도구를 실행했는지 추적 가능

3. 정책 팩 운영
- 부서별 허용 경로/도구/금지 명령을 정책 파일로 관리

4. 확장 게이트
- 기능 확장은 품질 게이트 통과 후에만 진행

## 7) 현실적 도입 시나리오 (Step 기반)
### Step 1. 업무 1종 고정
- 예: 회의록 입력 -> 업무 액션 리스트 생성 -> 검토 보고

### Step 2. 사용자 2군 분리
- 비IT 사용자군: 템플릿 기반 실행만 허용
- 운영 담당자군: 정책/로그/승인 관리

### Step 3. 검증 세트 운영
- 정상/실패/차단 3가지 시나리오를 표준 세트로 고정

### Step 4. 확장 판단
- 아래 기준 만족 시에만 다음 업무 템플릿 추가:
  - 정책 위반 0
  - 로그 누락 0
  - 승인 흐름 누락 0

## 8) 실패 가능성이 높은 패턴 (주의)
- 기능을 먼저 늘리고 정책을 나중에 붙이는 방식
- 비IT 사용자에게 자유 프롬프트를 강요하는 방식
- 승인 없는 자동 실행을 기본값으로 두는 방식
- 로그를 요약만 저장하고 실행 근거를 남기지 않는 방식

## 9) 최종 검증 의견
- 본 접근은 비IT 업무자 및 회사 대상 에이전트 전략으로 **충분히 접근 가능**하다.
- 단, 성공 확률은 아래 3요소에 결정된다:
  - 업무 템플릿화
  - 정책/권한/승인 내장
  - 테스트 게이트 기반 확장 통제

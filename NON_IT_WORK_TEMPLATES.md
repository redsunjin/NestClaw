# 비IT 사용자용 업무 템플릿 3종

## 공통 사용 규칙
- 자유 프롬프트 대신 템플릿 필드를 채워 실행한다.
- 민감한 개인 정보는 입력하지 않는다.
- 불확실한 실행은 승인 요청으로 전환한다.

## 템플릿 1) 회의요약 -> 액션리스트
### 입력
- 회의 제목:
- 회의 날짜:
- 참석자:
- 원문 메모:
- 보고 대상:

### 실행 지시
- 원문 메모를 핵심 논점 3~7개로 요약한다.
- 담당자/기한이 있는 액션 아이템으로 변환한다.
- 불명확한 항목은 `확인 필요`로 표시한다.

### 출력 형식
```markdown
# 회의 결과 요약
## 핵심 논점
- ...

## 액션 아이템
| 항목 | 담당자 | 기한 | 우선순위 | 상태 |
|---|---|---|---|---|
| ... | ... | ... | High/Medium/Low | Open |

## 확인 필요
- ...
```

## 템플릿 2) 주간 업무보고 초안
### 입력
- 보고 기간:
- 완료 업무:
- 진행 중 업무:
- 이슈/리스크:
- 다음 주 계획:
- 수신자:

### 실행 지시
- 완료/진행/리스크/계획을 분리해 간결하게 정리한다.
- 리스크는 영향과 대응안을 함께 제시한다.
- 수신자 기준으로 경영/실무 톤을 선택한다.

### 출력 형식
```markdown
# 주간 업무보고 (기간: ...)
## 1. 완료 업무
- ...

## 2. 진행 중 업무
- ...

## 3. 이슈 및 리스크
| 항목 | 영향 | 대응안 | 지원 필요 |
|---|---|---|---|
| ... | ... | ... | Yes/No |

## 4. 다음 주 계획
- ...
```

## 템플릿 3) 일정 정리 및 우선순위
### 입력
- 기간:
- 할 일 목록:
- 고정 일정:
- 마감일:
- 의존 관계:

### 실행 지시
- 할 일을 긴급도/중요도로 분류한다.
- 고정 일정과 충돌하지 않게 배치한다.
- 마감 위험이 있는 항목은 먼저 배치한다.

### 출력 형식
```markdown
# 일정 정리 결과 (기간: ...)
## 우선순위 매트릭스
| 업무 | 중요도 | 긴급도 | 권장 처리 순서 |
|---|---|---|---|
| ... | High/Medium/Low | High/Medium/Low | 1 |

## 권장 일정표
| 날짜 | 시간 | 업무 | 근거 |
|---|---|---|---|
| ... | ... | ... | ... |

## 주의 항목
- ...
```

## 운영 팁
- 처음에는 템플릿 1개만 활성화해 안정성 검증 후 확장한다.
- 템플릿 결과는 Reviewer를 통해 사실성/누락 여부를 점검한다.
- 승인 필요 항목(외부 발송, 민감 데이터 포함)은 자동 전송하지 않는다.
